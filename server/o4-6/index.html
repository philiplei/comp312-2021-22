4-x Database access

This lab demonstrates how to use a file-based database sqlite.

SQLite

SQLite is a software library that implements a self-contained, serverless, zero-configuration, transactional SQL database engine (from https://www.sqlite.org/). The whole database is stored in a file, and an application can open this database directly without connecting to any server.

A popular module to access SQLite in Node is sqlite3. Install this module and update package.json with the command npm install sqlite3 --save. Then create and initialize the sample database todo.sqlite with the program node init.js. That program runs the following SQL scripts to create a table and populate it.

Note: an INTEGER primary key in Sqlite is auto-increment.

DROP TABLE IF EXISTS `task`;
CREATE TABLE IF NOT EXISTS `task` (
  `id` INTEGER PRIMARY KEY,
  `due` TEXT NOT NULL,
  `what` TEXT NOT NULL,
  `category` TEXT NOT NULL,
  `done` INTEGER NOT NULL
);

  ('2016-10-29', 'Buy bday gift', 'home', 1),
  ('2016-10-31', 'Mary''s birthday', 'home', 0),

Retrieve data with SELECT

SQLite supports both basic and advanced features of SELECT statement. The following demonstrates the steps to read data from a database.

var sqlite3 = require('sqlite3').verbose();
// open the database at the given file name
var db = new sqlite3.Database("todo.sqlite");

// retrieve result set 1 row at a time
db.each("SELECT * FROM task", function(err, row) {
  if (err) throw err;
  console.log(row.due + ": " + row.what);
});

// retrieve complete result set in an array of rows
db.all("SELECT * from task", function(err, rows) {
  if (err) throw err;
  console.dir(rows);
})

// close the database
db.close();

After importing the module sqlite3, specify the file name of the database and open it with the constructor sqlite3.Database(). Use the methods db.each() or db.all() to run a SELECT query. The callback to these methods have two parameters (similar to file I/O functions is fs). The first parameter is error. If the error is null, the SQL runs successfully, and the result is available in the second parameter.

The first method db.each() returns the result set row by row. Each row is represented by a JavaScript object, with one property for each column. This is suitable for reading data from a large result set.

The second method db.all() returns the complete result set in an array rows. This is more convenient, and easier to use for relatively small result set.

You might notice that we use the function expression in callback. This is required because sqlite3 uses the this keyword for result in UPDATE, DELETE and INSERT. For consistency, we’ll use function expression instead of arrow functions in all callbacks for our SQLite examples.

Placeholders

You can supply parameters to fill in placeholders in SQL statements in sqlite3. This is especially useful because the module will perform suitable quoting for string. Please refer to online reference for more ways to use placeholders.

// show due date and task description for completed tasks in the study category
db.each("SELECT due, what FROM task WHERE done=? AND category=?",
  [1, 'study'],
  function(err, row) {
    if (err) throw err;
    console.log(row.due + ": " + row.what);
  }
);

INSERT, UPDATE and DELETE

Use the method db.run() to execute INSERT, UPDATE and DELETE statements. In the callback to this method, the property this.changes indicates the number of rows affected by the query, and the property this.lastID contains the value of the last inserted row ID. (That is the INTEGER PRIMARY KEY in our table. Refer to this for more info.)

db.run('INSERT INTO task (due, what, category, done) VALUES (?,?,?,?)',
  [ '2016-11-11', 'comp312 test', 'study', 0 ],
  function(err) {
    if (err) throw err;
    console.log('id of the new row = ', this.lastID);
  }
);

Next is an example of DELETE.

// delete all completed tasks
db.run('DELETE FROM task WHERE done=1',
  function(err) {
    if (err) throw err;
    console.log('Number of rows deleted = ', this.changes);
  }
);
