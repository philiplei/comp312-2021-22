4-7 Web app middlewares

This lab demonstrates how to use middleware in general. It introduces a few common middleware, and also covers a middleware that handles form submit using GET and POST.

Middlewares

In the previous lab, we saw that an Express app is essentially a sequence of routes. When a request arrives at an Express app, the app tries to find a match of the path in the order of routes in the sequence. If there is a match, the app assumes the callback of the route will generate a response, and skip any remaining routes.

The above notion is simplification. An Express app is in fact a sequence of middlewares and routes. An incoming request will go through the middlewares and routes in sequential order until a response is generated.

Middlewares are JavaScript functions in Express. Most middlewares are not included in Express module, and must be installed using npm install or package.json separately. Install the middlewares and Express with npm install before running the example app3.js, which demonstrate the basic structure of an Express app with middlewares.

// app3.js
// middlewares

var express = require('express');
var app = express();

// import a middleware, which logs the request
var morgan = require('morgan');
// install the middleware into the app
app.use(morgan('short'));

// import another middleware, which secures the app with HTTP headers
var helmet = require('helmet');
// install the middleware
app.use(helmet());

// routes ...

app.get('/', (req, res) => {
  res.send('Hello world');
});

app.get('*', (req, res) => {
  res.status(404).send('No match');
});

// start listening at TCP port 3000
app.listen(3000);

When the app receives an HTTP request, the first middleware ‘morgan’ prints a log message in the console. Next, the middleware ‘helmet’ adds some HTTP headers (e.g. X-XSS-Protection) to the response to secure the app. Afterwards, the request is checked against each route in sequence.

Most middlewares should be registered before routes. If a middleware is registered after a route, and the route returns a response for a request, the middleware will NOT run for this request. As an experiment, move the middleware ‘morgan’ between the two routes.

Serving static content

There is a special middleware bundled with Express. This middleware express-session serves static assets (typically HTML, CSS, images, and client-side library) to clients. The middleware tries to match the files in a folder against the URL path of the request. If there is a match, it returns the file, with a suitable MIME-type, in a response. This makes it simple to creates a simplistic HTTP server.

// excerpt from app4.js
var express = require('express');
var app = express();

app.use(express.static(__dirname + '/public'));

app.listen(3000, () => {
  console.log('Server running at http://localhost:3000/');
});

In general, routes should be placed before the ‘static’ middleware. Otherwise, a static file that matches the URL path will be handled by the ‘static’ middleware and will not be handled by any routes after the middleware. Try to move the specified route in app4.js to experiment.

Handling form submit

Data that are submitted using GET method are available in the object req.query. In the following example, the route GET ‘/add’ obtains the two numbers a and b from a GET request, and returns the result of ‘a+b’ in response.

// app5.js

var express = require('express');
var app = express();

app.get('/form1.html', (req, res) => {
  res.send(`<!doctype html>
<html>
<body>
  <p>Enter 2 numbers to add:</p>
  <form action='/add' method='GET'>
    <p>a: <input name='a' type='text'/>
    <p>b: <input name='b' type='text'/>
    <p><button type='submit'>Calculate a+b</button>
  </form>
</body>
</html>`);
});

app.get('/add', (req, res) => {
  var a = parseFloat(req.query.a);
  var b = parseFloat(req.query.b);
  res.send(`${a} + ${b} = ${a+b}`);
});

You’ll need a middleware to extract data from the message body of a POST request. A popular choice is the middleware body-parser. This middleware parses the request message body (if any) and populate the object req.body. In the route in the sample code below (see full source at app6.js), data submitted in POST request are accessible in req.body.mesg.

// excerpt from app6.js

var bodyParser = require('body-parser');
app.use(bodyParser.urlencoded({extended:false}));

app.post('/append', (req, res) => {
  var mesg = req.body.mesg;
  fs.appendFile(__dirname+'/log.txt', mesg+'\n', 'utf8', (err)=>{
    if (err) throw err;
    res.send(`Append successful.
      You can <a href="/getlog">retrieve the log</a> now.`);
  })
});
